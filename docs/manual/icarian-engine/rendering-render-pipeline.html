RenderPipeline

Namespace: IcarianEngine.Rendering

Description

RenderPipelines are used to determine how a render pass is handled. This runs on a separate thread(s) from the main thread. Each camera uses their own group of threads.

Functions

abstract void Resize ( uint a_width, uint a_height )

Called when the swapchain is resized. Runs on the main render thread.

abstract void ShadowSetup ( Camera a_camera )

Called before the shadow pass. Runs on shadow thread(s) in ThreadPool.

abstract Matrix4 PreShadow ( Light a_light, Camera a_camera, uint a_textureSlot )

Called before the shadow pass for a particular texture. Returns LVP matrix. Runs on shadow thread(s) in the ThreadPool.

abstract void PostShadow ( Light a_light, Camera a_camera, uint a_textureSlot )

Called after the shadow pass for a particular texture. Runs on shadow thread(s) in the ThreadPool.

abstract void PreRender ( Camera a_camera )

Called before the main deffered render pass. Runs on render thread(s) in the ThreadPool.

abstract void PostRender ( Camera a_camera )

Called after the main deffered render pass. Runs on render thread(s) in the ThreadPool.

abstract void LightSetup ( Camera a_camera )

Called before the lighting pass. Runs on lighting thread(s) in the ThreadPool.

abstract Material PreLight ( LightType a_lightType, Camera a_camera )

Called before the lighting pass for a particular light type. Returns the material to use for the lighting pass. Runs on lighting thread(s) in the ThreadPool.

abstract void PostLight ( LightType a_lightType, Camera a_camera )

Called after the lighting pass for a particular light type. Runs on lighting thread(s) in the ThreadPool.

abstract void PostProcess ( Camera a_camera )

Called to do post processing after all passes and before UI. Runs on own thread(s) in the ThreadPool.

static void Init ( RenderPipeline a_pipeline )

Set which RenderPipeline to use. Uses the default if not called.
